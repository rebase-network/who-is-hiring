职责描述

  • 交易系统架构负责人
  • 设计并落地高性能交易系统整体架构（撮合引擎 / 永续合约 / 风控 / 清算）
  • 构建低延迟、高吞吐、高一致性的核心交易基础设施（对标CEX级性能）
  • 核心引擎与性能优化
  • 深度参与撮合引擎设计与优化（内存撮合、并发控制、撮合一致性）
  • 主导系统级性能优化（延迟、吞吐、稳定性、极端行情处理）
  • 解决关键技术问题：如撮合一致性、资金安全、强平机制、风险隔离等
  • 自研链 × 交易系统一体化
  • 参与或主导自研链架构设计（执行层 / 共识 / 数据结构）
  • 优化交易系统与链的集成路径（链上/链下撮合、结算机制、状态同步）
  • 提升整体系统性能（latency、finality、吞吐）
  • 技术决策与工程体系
  • 主导技术选型、架构演进与风险评估
  • 建立高标准研发体系（代码质量、评审机制、发布流程）
  • 提升团队工程能力与系统稳定性
  • AI驱动工程效率（加分）
  • 推动AI在研发效率、性能分析、风控等场景的落地

任职要求

  • 有交易系统核心模块实战经验（撮合引擎 / 永续合约 / 风控）；深入理解低延迟交易系统架构（内存撮合、事件驱动、并发模型）；有高性能系统优化经验（延迟 / 吞吐 / 一致性）；熟悉 Rust / C++（至少精通一种）；有0到1或大规模重构经验。加分：自研链或交易链经验、链上/链下撮合结算与状态同步、Hyperliquid/CEX底层架构、高性能中间件（Kafka/Redis/Aeron）、极端行情高并发处理。
